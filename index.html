    Introduction

    Welcome to the Liopa LipSecure API!

    We have language bindings in cURL, with Python and Java to follow soon. You can view code examples in the dark area to the right, and you can switch the programming language of the examples with the tabs in the top right.

    Authentication

    To authorize, use this code:

    # With shell, you can just pass the correct header with each request
curl -X POST “https://beingbuilt.lipsecure.com/LiopaAPI/api/challenge”
  -H  “X-API-Key: 38332b622aea11e8b103a860b60304d5”
  [remaining payload]

    Make sure to replace 38332b622aea11e8b103a860b60304d5 with your API key.

    LipSecure uses API keys to allow access to the API, by invitation only.

    LipSecure expects for the API key to be included in all API requests to the server in a header that looks like the following:

    X-API-Key: 38332b622aea11e8b103a860b60304d5

    You must replace 38332b622aea11e8b103a860b60304d5 with your personal API key.

    Phrase Verifier

    Get a random challenge phrase

    curl -X POST "https://beingbuilt.lipsecure.com/LiopaAPI/api/challenge" -H  "accept: application/json" -H  "X-API-Key: 38332b622aea11e8b103a860b60304d5" -H  "content-type: application/json" -d ""{}""

    The above command returns JSON structured like this:

      {
      "id": "ef5a0ae2-884e-4106-b247-b332a0f9dd88",
      "phrase":"597692",
      "error":
        {
            "type":"",
            "code":0,
            "message":"OK"
        }
    }

    This endpoint requests a random phrase from the LipSecure Phrase Verifier, returning the phrase and a transaction ID. The transaction ID will be used in subsequent calls.

    HTTP Request

    POST https://beingbuilt.lipsecure.com/LiopaAPI/api/challenge

    Verify a recorded phrase

    curl -X POST "https://beingbuilt.lipsecure.com/LiopaAPI/api/verifier/ef5a0ae2-884e-4106-b247-b332a0f9dd88"
    -H  "accept: application/json"
    -H  "X-API-Key: 38332b622aea11e8b103a860b60304d5"
    -F ‘upload=@<video_file>’

    The above command returns JSON structured like this:

    {
    "id" : "ef5a0ae2-884e-4106-b247-b332a0f9dd88",
    "phrase" : "597692",
    "confidence":84,
    "error" :
        {
            "type" : "",
            "code":0,
            "message":"OK"
        }
}

    This endpoint verifies the phrase in the provided video.

    HTTP Request

    POST http://lipsecure.com/api/verifier/<id>

    URL Parameters

    Parameter Description
    id The transaction ID, that must have been produced by the server using the preceeding GET request

    Video File Payload

    The payload should be a compressed video file, containing a subject reciting the phrase. Resolution, frame rate, bitrate etc are all flexible. As a guideline, use MPEG-4 encoding of 640x480 video at 15 frames-per-second with a 1kbps bitrate.

    Errors

    The LipSecure Service will return JSON payloads containing error details. The JSON format for errors is as follows: json "error" : { "type" : # one of 'client', 'server' or 'user', "code": <id>, "message": # string with description }

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    500 Internal Server Error -- Contact support@liopa.co.uk
